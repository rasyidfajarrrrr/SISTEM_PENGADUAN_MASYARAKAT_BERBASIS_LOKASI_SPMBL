SPMBL - Sistem Pengaduan Masyarakat Berbasis Lokasi PTI

Rasyid Fajar Pastika | NIM: A710230050 | RPL 44

1. Pendahuluan

1.1 Tujuan

Dokumen ini menjabarkan spesifikasi kebutuhan perangkat lunak untuk SPMBL yang bertujuan untuk menangani laporan masyarakat berdasarkan lokasi secara efisien dan aman.

1.2 Lingkup Sistem

Sistem berbasis web dan mobile untuk membuat laporan kejadian dengan foto, deskripsi, dan lokasi. Petugas dapat menangani laporan, admin mengelola data pengguna dan statistik.

1.3 Definisi, Akronim, dan Singkatan

  • SPMBL: Sistem Pengaduan Masyarakat Berbasis Lokasi
  • GPS, API, JWT, UI, UX, CRUD

2. Deskripsi Umum

2.1 Perspektif Produk

  • Frontend: React/Vue dan Flutter
  • Backend: Node.js atau Laravel
  • Database: PostgreSQL/MySQL
  • Integrasi: Google Maps API, FCM, layanan email

2.3 Karakteristik Pengguna

PeranKeterangan
PenggunaMasyarakat umum pelapor kejadian
PetugasPersonel yang memproses laporan
AdminPengelola dan pengawas sistem

3. Kebutuhan Fungsional

  • Registrasi dan login
  • Pembuatan dan pemrosesan laporan
  • Manajemen pengguna dan statistik
  • Notifikasi perubahan status laporan

4. Kebutuhan Non-Fungsional

  • Responsif, cepat, aman, log dan backup otomatis
  • UI ramah pengguna

5. Perencanaan Sprint

2Lokasi otomatis/manual, gambar, peta
3Dashboard petugas dan status
